This file is the source of truth for Claude Code when working on this project. Read it fully before making any changes.

## Project Purpose

A local Node.js CLI tool that automates applying to ~80+ German master's programs for Winter Semester 2026/27. It tracks programs, generates tailored motivation letters via the Claude API, manages documents, and surfaces daily action plans.

---

## Architecture

### Runtime
- **Node.js** (ESM modules — `"type": "module"` in package.json)
- **Entry point**: `src/index.js` — registers all commands via `commander.js`
- **No web server, no database** — all state lives in JSON files under `data/`

## Coding Rules

### General
- ESM only — use `import`/`export`, never `require()`
- Always `await` async operations — no unhandled promise rejections
- Every command must produce meaningful terminal output — never silent success
- Never crash on missing optional data — degrade gracefully
- All JSON written with `JSON.stringify(data, null, 2)`

### File I/O (via `src/lib/database.js`)
- All reads/writes go through `database.js` helpers, not raw `fs` calls in commands
- `readJSON(path)` returns `null` (not throws) if file doesn't exist
- `writeJSON(path, data)` creates parent directories automatically

### Error handling
- API errors → write to `output/errors.log` and continue (especially in batch)
- Missing profile → print `"Profile incomplete. Run: node src/index.js setup init"` and exit 1
- Missing programs.json → print `"No programs found. Run: node src/index.js import <file>"` and exit 1

### Terminal output (via `src/lib/formatter.js`)
- Use `chalk` for colors — never raw ANSI codes
- Use `cli-table3` for tables
- Use `ora` for spinners during async work
- Use box-drawing characters for summary dashboards
- Color conventions:
  - Red = urgent / missed / error
  - Yellow = warning / filling / approaching deadline
  - Green = accepted / done / healthy
  - Cyan = pending / info

### Anthropic API (via `src/lib/anthropic.js`)
- Model: `claude-sonnet-4-20250514`
- `max_tokens`: 1200 for letters, 600 for scoring
- Retry on HTTP 529 with 5s backoff, max 3 retries
- Load key from `process.env.ANTHROPIC_API_KEY` (dotenv loaded in `src/index.js`)
- 2-second delay between calls in batch mode

## Key Implementation Notes

### Yellow cell detection (Uni-Assist)
In `exceljs`, check `cell.fill?.fgColor?.argb`. Yellow highlight = `'FFFFFF00'`. Use:
```js
const isYellow = cell.fill?.fgColor?.argb?.startsWith('FFFF') ?? false;
```

### Deadline parser
`src/lib/deadlineParser.js` must handle all formats:
- `"31 May"` → `2026-05-31`
- `"15 May/15 Jul"` → `2026-05-15` (first date)
- `"1 Apr to 31 May"` → `2026-05-31` (end date, treat as latest)
- `"25.05 to 18.09"` → `2026-05-25` (start date)
- `"15 Jul, 15 Aug - 31 Aug"` → `2026-07-15` (earliest)
- Excel date serial numbers → use exceljs date conversion
- Return `null` for unparseable strings — never throw

### Eligibility flags (keyword matching on `description` + `notes`)
- `ECTS_GAP` — contains "210 ECTS", "240 ECTS", "Bachelor with 210", etc.
- `GERMAN_REQUIRED` — language field is German with no English track, or contains "Deutschkenntnisse", "C1", "nur auf Deutsch"
- `DEADLINE_PASSED` — `deadlineWinterParsed` < today
- `GPA_RISK` — contains "competitive", "numerus clausus", "GPA above"
- `STRONG_FIT` — English language, ≥180 ECTS ok, tuition-free, deadline not passed

### Priority scoring (0–100)
| Factor | Weight |
|--------|--------|
| English program | +20 |
| ECTS ≥ 180 acceptable | +25 |
| GPA match (≤ 2.5 German scale) | +20 |
| Deadline not missed | +15 |
| Tuition-free | +5 |
| Ranking bonus | +10 |
| Uni-Assist penalty | -5 |
| German required | -20 |

### Import idempotency
If `programs.json` exists, confirm before overwriting. Merge on `id` to preserve manually-set fields: `status`, `notes`, `appliedDate`, `documentsSubmitted`, `motivationLetterGenerated`, `motivationLetterPath`, `letterVersions`.

### Startup warning hook
On every command invocation, after loading programs, check if any program has `deadlineWinterParsed` within 7 days and status is not `missed`/`accepted`/`rejected`. If yes, print a compact warning box before the command output.

## Prompt Engineering (Letter Generator)

**System prompt:**
> You are a graduate admissions consultant helping a Moroccan CS student write motivation letters for German master's programs. Write authentic, specific letters that sound human — not AI-generated. Avoid clichés. Be direct. Each letter must be tailored to the specific program and university. Never reuse phrasing from one letter to another.

**Banned phrases** (enforce in prompt):
`"I am passionate about"`, `"ever since I was young"`, `"in today's rapidly evolving world"`, `"I am confident that"`, `"it would be an honor"`, `"I have always been fascinated"`

**Letter structure** (enforce in prompt):
1. Opening — why THIS program at THIS university (reference something unique)
2. Academic background — specific relevant coursework from Moroccan CS degree
3. Projects — 1–2 GitHub projects tied to program focus
4. Why Germany — brief and genuine
5. Career vision — concrete post-graduation goals

**Do NOT start the letter with:** `"I am writing to apply..."`
